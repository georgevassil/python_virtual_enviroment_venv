# 🐍 Οδηγός: Python Virtual Environment (venv)

## 💡 Πώς Λειτουργεί (Σημαντική Σημείωση)
Το εικονικό περιβάλλον (venv) είναι ένας φάκελος που λειτουργεί σαν **απομονωμένο "sandbox" αποκλειστικά και μόνο για την Python**. 

**Τι πρέπει να γνωρίζεις:** * **Απομονώνει ΜΟΝΟ:** Την έκδοση της Python και τις βιβλιοθήκες (packages) που εγκαθιστάς μέσω `pip` για το συγκεκριμένο project. Αποτρέπει τις συγκρούσεις εκδόσεων (version conflicts) μεταξύ διαφορετικών projects.
* **ΔΕΝ απομονώνει:** Το λειτουργικό σύστημα, τα αρχεία του υπολογιστή σου ή άλλα εξωτερικά προγράμματα/υπηρεσίες. Δεν είναι Virtual Machine ούτε Container (π.χ. Docker).
* **Συνδέεται με το Τερματικό, ΟΧΙ με τον φάκελο:** Μόλις ενεργοποιήσεις το venv, παραμένει ενεργό για όλη τη συνεδρία του τερματικού. Ακόμα κι αν αλλάξεις φάκελο (με `cd`), το venv σε "ακολουθεί" μέχρι να το κλείσεις ρητά (με `deactivate`).

---

## 🚀 Βασικές Εντολές (Cheat Sheet)

### 1. Δημιουργία Περιβάλλοντος
Άνοιξε το τερματικό σου στον φάκελο του project και τρέξε την κατάλληλη εντολή:

* **Windows:** `python -m venv venv`
* **macOS / Linux:** `python3 -m venv venv`
*(Ο φάκελος που θα δημιουργηθεί θα ονομάζεται `venv`).*

### 2. Ενεργοποίηση (Activation)
Πρέπει να το ενεργοποιείς **κάθε φορά** πριν τρέξεις τον κώδικα ή εγκαταστήσεις νέα πακέτα:

* **Windows (Command Prompt):** `venv\Scripts\activate.bat`
* **Windows (PowerShell):** `venv\Scripts\Activate.ps1`
* **macOS / Linux:** `source venv/bin/activate`
*(Όταν ενεργοποιηθεί, θα δεις την ένδειξη `(venv)` στην αρχή της γραμμής του τερματικού σου).*

### 3. Διαχείριση Βιβλιοθηκών (Packages)
Έχοντας ενεργοποιημένο το venv, μπορείς να διαχειριστείς τα πακέτα σου με ασφάλεια:

* **Εγκατάσταση νέου πακέτου:** `pip install <όνομα_πακέτου>`
* **Αποθήκευση όλων των πακέτων σε αρχείο (για να το ανεβάσεις στο GitHub):** `pip freeze > requirements.txt`
* **Εγκατάσταση όλων των πακέτων (αν κατέβασες το project από το GitHub):** `pip install -r requirements.txt`

### 4. Απενεργοποίηση (Deactivation)
Όταν τελειώσεις τη δουλειά σου, απλά γράψε (σε οποιονδήποτε φάκελο κι αν βρίσκεσαι):

* `deactivate`
